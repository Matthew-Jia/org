
# Linker.lds

By default, a linker just piles code and date together in the order it finds them. For an operating system, this ``random" pile is fatal. 




# Global keyword

In NASM, the global keyword is a linker directive. It tells the assembler to export a specific symbol so that it is visible to the outside world. When you use the global keyword, you are essentially creating a ``public API" for that file.
